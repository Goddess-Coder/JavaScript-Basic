JavaScript Basics

JavaScript is a cross-platform, objecti-oriented scripting lauguage.JavaScritp is extremely popular for a varity of reasons. It is a small and lightweight language allowing maximum flexibility for developers to take it in a bunch of different directions. JavaScript lives inside a host enviroment a webbrowser or Node server, it can be connected to the objects of these environments to provide programmactic control over them.

  • Variable Declaration JavaScript variables are containers for storing data values - imagine a cup you fill with coffee, the cup holds the coffee, a variable holds a value. All JavaScript variables must be identified with unique names are called identifiers. var x;
  • Variable Assignment Assignment operators assign values to JavaScript variables - our cup can not have coffee poured in it, giving your variable a value to hold. the = assignment operator assigns a value to a variable. var x = 10;
  • Data TypesData Types are an important concept; to be able to operate on variables you need to know the data type. There are six data types that are JavaScript primitives: Boolean - true or false; null- null aka nothing; Number - 42 or 3.14159; String - "Coding Dojo Rocks!"; Array - [1,'Coding', 2, 'Dojo']; and Object - {first_name: 'Jane',last_name:'Doe'}
  • Conditionals When you write code, you want to perform different actions for different decisions There are the following conditional statements: if a specified condition is true, do this code in our code block; else if to specify a new condition to test, if the first condition is false; else we execute this block of code
  • LoopsThere are many different kinds of lopps in every programming language, but they all essentially do the same things: they will repeat an action some number of times. Imagine you gave to run a mile, well you run around the track four times and then you stop. That's a loop!
  • Functions Functions are an enscaspulation of a code block. When we call our function this will run that code block. Think of it as a list of instructions. s an example imagine we are putting together a desk for Ikea, we open up the instruction manual and get started, first we screw the legs to the table top; next we place the table the right way up. Done! We finished our instructions. Sadly there are a ton more tables to do so lets call our function over and over and over again.
